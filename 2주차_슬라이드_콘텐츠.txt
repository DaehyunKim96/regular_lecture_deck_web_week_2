[슬라이드 1]
제목: 여러분이 원했던 자동화, 모아보니

부제: 사전 설문 응답 정리


[슬라이드 2] 설문에서 나온 자동화 요청 리스트

- 데이터 추출 후 엑셀 대시보드로 정리
- 특정 분야 기관·단체의 세미나/포럼/행사 일정을 웹에서 조사해 정기적으로 보고
- 서류(논문 등)와 심사자 프로필을 비교해 적합한 담당자를 매칭
- 회의 메모를 회의록 양식으로 정리하고 업무보고로 재구성
- 주간 계획표 작성, 일정 조율, 메시지 확인 및 할 일 정리
- 여러 자료를 취합해 정해진 양식의 문서로 완성
- 메일함에 쌓이는 정보성 메일을 분류하고 정리
- 메일·문서에서 필요한 정보만 뽑아서 정리


[슬라이드 3] 겉보기엔 다 다른 업무, 그런데...

- 업종도, 다루는 자료도 제각각
- 그런데 뜯어보면 거의 같은 순서로 움직인다


[슬라이드 4] 공통 구조: 4단계

1. 수집 (Input) — 메일함, 웹, 문서에서 자료를 모은다
2. 추출/분류 (Process) — 기준에 따라 필요한 정보만 골라낸다
3. 산출물 생성 (Output) — 엑셀, 문서, 보고서 형태로 정리한다
4. 공유/배포 (Delivery) — 보고하거나, 메신저·협업툴로 전달한다

→ 여러분이 적어주신 자동화 요청은 대부분 이 네 단계 중 하나 이상에 해당


[슬라이드 5] 왜 AI Agent와 잘 맞는가

- 4단계 모두 "정해진 규칙 + 반복 작업"의 성격
- 사람이 매번 판단해야 할 것 같지만, 실제로는 기준을 한 번 정의하면 반복 가능
- AI Agent = 이 기준을 프롬프트로 정의해주면, 도구(웹검색/파일읽기/문서생성)를 스스로 골라 4단계를 대신 수행하는 존재


[슬라이드 6] 오늘 실습에서 다룰 것

- 4단계 중 "추출 → 산출물 → 공유" 구간을 직접 만들어본다
- 대상 업무: 회의 메모 → 회의록 → 업무보고 (→ 블로그 초안)
- 이후 이 워크플로우를 반복 자동화(예약 작업)로 전환
