# 설문조사 분석 및 강의 기획안 (2주차)

## 1. 설문 응답 요약

### 반복적이고 시간이 많이 드는 업무 (Pain Point)

| 구분 | 내용 |
|---|---|
| 데이터/보고 | 데이터 추출 후 엑셀 대시보드화 |
| 일정 리서치 | 국회·국방부·안보 관련 연구기관/대학/언론사의 세미나·포럼·전시회 등을 웹서핑으로 조사해 주 2회 일정표 작성 및 보고 |
| 매칭 업무 | 학술지 투고논문 심사위원 섭외 매칭 (교수 약력 기반) |
| 문서 작업 | 회의록 작성, 업무보고 |
| 일정 관리 | 주간 계획표 작성, 시수 맞추기, 메시지 확인 및 할 일 정리 |
| 문서 정리 | 자료취합 및 고퀄리티 한글(HWP)파일 정리 |
| 메일 처리 | (개인) 네이버 메일 정보성 메일 분류·정리, (업무) 사내 메일·문서에서 특정 정보 추출 |

**공통 패턴:** 대부분의 업무가 "① 정보 수집(웹서핑/메일/문서) → ② 특정 기준으로 분류·추출 → ③ 정리된 산출물(엑셀·문서·보고서) 생성 → ④ 공유·보고" 흐름을 갖는다. 이는 AI Agent(Claude/Codex)가 강점을 갖는 전형적인 워크플로우와 일치한다.

### 원하는 최종 결과물

- 쿼리 자동 실행 + 시각적 대시보드화 + Slack 자동 게시 (전 과정 자동화)
- 웹 크롤링 기반 대시보드 (예: 백화점 특정 브랜드 신규 등록/인기상품/증정품/가격 정보)
- **일정 보고 실시간 업무자동화 (응답 1순위)**
- 회의록 작성 → 업무보고 → 블로그 자동 작성
- 반복 업무를 원하는 방식으로 처리하는 자동화 시스템 구축
- 사내 템플릿 기반 고퀄리티 PPT 작업
- 한글파일 작업 자동화
- 이메일/텍스트 문서 → 정의된 기준으로 분류·특정정보 추출 → 정리된 결과물

### 강의에 기대하는 점

- AI 업무자동화를 실제로 "도입"해보는 경험
- 회의록 → 업무보고 → 블로그 자동 작성 같은 구체적 사례
- 실제 현업에서 쓰이는 다양한 활용 예시
- 스스로 생각지 못했던 자동화 사례(Case Study)를 간접 체험

## 2. 강의에서 설명할 내용 (개념 파트)

1. **AI Agent의 작동 원리** — Claude/Codex가 웹검색, 파일 읽기/쓰기, 코드 실행 같은 "도구"를 스스로 선택해 작업을 수행하는 원리를 비개발자 눈높이에서 설명. "명령을 실행하는 챗봇"이 아니라 "일을 대신 처리하는 직원"이라는 프레이밍.

2. **반복업무 자동화의 공통 구조** — 수집(Input) → 처리/추출(Process) → 산출물 생성(Output) → 배포/공유. 설문에 나온 사례 대부분이 이 4단계 중 어디에 해당하는지 매핑해서 보여주기.

3. **웹 리서치 자동화** — 특정 조건(기관, 키워드, 기간)으로 웹을 검색·수집해 표 형태로 정리하는 법. 세미나 일정 조사, 크롤링 기반 상품 정보 수집 사례와 직결.

4. **문서·이메일에서 정보 추출/분류** — 여러 개의 이메일이나 텍스트 문서를 업로드했을 때, 정해진 기준(분류 카테고리, 추출 항목)을 프롬프트로 정의해 자동 정리하는 법.

5. **엑셀/대시보드 자동 생성** — 정리된 데이터를 표, 피벗, 차트로 변환하는 법. "대시보드"가 실무에서는 결국 잘 구조화된 엑셀 파일 + 시각화라는 점을 짚기.

6. **문서 자동 생성 (Word/PPT/한글)** — 회의록, 업무보고, 사내 템플릿 기반 PPT를 만드는 법. 한글(HWP) 파일은 AI 도구가 직접 편집하기 까다로운 포맷이므로, Word로 대체 작성 후 변환하거나 사내 양식을 Word/Excel 기반으로 전환하는 우회 전략을 함께 안내.

7. **정기 자동화(스케줄링) 개념** — "매주 2회 보고", "매일 아침 메일 정리"처럼 반복 주기가 있는 업무를 자동 실행으로 전환하는 개념. 지금 당장은 사람이 트리거해야 하는 부분과, 스케줄로 완전 자동화 가능한 부분을 구분해서 설명.

8. **결과물 공유 자동화** — Slack, 블로그 등 외부 채널에 결과를 자동 게시하는 개념(커넥터/연동의 개념 수준 소개, 계정 연동이 필요한 부분은 데모 위주로).

9. **좋은 프롬프트 설계법** — 반복업무를 AI에게 위임할 때 "무엇을, 어떤 기준으로, 어떤 형식으로" 요청해야 원하는 결과가 나오는지. 분류 기준표를 먼저 만들고 프롬프트에 반영하는 습관 강조.

## 3. 실습 내용 (난이도순)

### 실습 1. 이메일/문서 → 엑셀 자동 정리 (기초, 전원 대상)
- 여러 건의 메일 또는 텍스트 문서를 업로드
- 분류 기준(카테고리, 추출 항목)을 함께 정의
- Claude가 기준에 따라 분류·추출해 엑셀 표로 정리
- 대응 니즈: 네이버 메일 분류, 사내 메일·문서 정보 추출, 자료취합

### 실습 2. 웹서핑 기반 일정표 자동 작성
- 특정 기관/키워드(세미나, 포럼, 전시회 등)로 웹 검색
- 결과를 일정표(날짜, 기관, 행사명, 링크) 형태로 정리
- 요약 보고 문서까지 자동 생성
- 대응 니즈: 응답 1순위였던 "일정 보고 실시간 자동화", 국회·국방부 세미나 리서치

### 실습 3. 회의록 → 업무보고 자동 변환
- 회의 메모/녹취 텍스트를 입력
- 정해진 회의록 양식으로 자동 정리
- 업무보고 문서로 재구성 (확장 실습: 블로그 포스트 초안까지 생성)
- 대응 니즈: 회의록 작성 업무보고, 블로그 자동 작성

### 실습 4. 사내 템플릿 기반 PPT/문서 자동 생성
- 기존 사내 템플릿(PPT 또는 Word)에 데이터를 채워 넣어 완성본 생성
- 대응 니즈: 사내 템플릿 고퀄 PPT, 한글파일 정리 (Word 대체 전략 시연)

### 실습 5 (심화, 시간 여유 시). 정기 자동화 + 결과 공유 데모
- 실습 1~2에서 만든 워크플로우를 스케줄 기능으로 "매일/매주 자동 실행"으로 전환
- 크롤링 결과를 엑셀 대시보드로 시각화하고, Slack 연동 데모로 자동 게시까지 시연
- 대응 니즈: 쿼리 자동화 + 대시보드 + Slack 게시, 반복업무 자동화 시스템 구축

### 보너스 케이스 스터디 (시간 되면 소개만)
- 논문 심사위원 매칭: 텍스트로 된 교수 약력 목록과 논문 주제를 주고, 적합도 기준으로 매칭·순위화하는 사례 — "정보 매칭·판단" 유형 자동화의 예시로 소개

---

## 4. 진행 제안

같은 재료(메일함/문서/웹검색)로 실습 1→2→3을 순서대로 진행하면 "수집 → 추출 → 산출물"이라는 하나의 파이프라인을 완성하는 경험을 줄 수 있음. 마지막에 실습 5로 "이걸 매주 자동으로 돌린다면?"을 보여주면 설문에서 반복 등장한 "실시간/정기 자동화" 기대치에 정확히 부합.
